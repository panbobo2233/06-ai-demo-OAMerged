## 三、定制开发（重新分期）

### P0（必须一期交付）

| 功能 | 理由 | 会议诉求 |
|------|------|----------|
| 界面优化，UI视觉效果更新 | Demo已经在做了 | #21 首页嵌套深 |
| 个性化主题、多主题切换、自定义布局 | 领导视图/标准视图已开发 | |
| 后台自定义配置 | 运维不能每次都改代码 | |
| 物理机→虚拟服务器+NVMe SSD | 部署前提条件 | #20 系统稳定性 |

### P1（二期后续交付）

| 功能 | 理由 | 会议诉求 |
|------|------|----------|
| 冗余流程剔除或合并 | 需业务部门配合梳理 | |
| 代码、算法优化 | 能跑起来再优化 | #7 发文查询慢 |
| 缓存优化（Redis） | 量上来了再做 | #7 #31 性能问题 |
| 数据库优化/迁移 | 同上 | |
| 定制化工作台和数据视图 | 流程引擎跑通后做 | #4 管理驾驶舱 |
| 低代码开发（升级版） | 标准版先用，不够再升 | #5 低代码需求 |
| 开放集成（外接AI） | 厂商AI智能体先用 | |
| 集成连接器 | 对接需求明确了再做 | |

### P2（二期后期或三期）

| 功能 | 理由 |
|------|------|
| 技术架构扩展（微服务拆分） | 工程量最大、风险最高 |
| 数据库读写分离、分库分表 | 有了量才需要 |
